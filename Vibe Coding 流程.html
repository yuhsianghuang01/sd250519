  • 以自然語言表達您的想法。
  • 不必過於詳細，重點在於傳達核心概念。
  • 可以提及目標技術棧 (例如：使用 React、Python Flask)。

2. 提示產生程式碼：描述期望的功能 將任務分解為更小、更具體的指示。
  • 逐步引導 Copilot 實作每個功能。
  • 使用清晰且明確的語言。
  • 可以提供輸入/輸出範例或所需的資料結構。
  • 指定程式語言、函式庫或框架。

3. 審查與接受：檢查產生的程式碼 仔細檢查 Copilot 產生的程式碼。檢查正確性、潛在的錯誤和安全漏洞。在接受之前先理解產生的程式碼。
  • 仔細閱讀每一行程式碼。
  • 檢查是否有語法錯誤、邏輯錯誤或潛在的安全風險。
  • 使用 Copilot 的 /explain 功能來理解不熟悉的部分。
  • 考慮單元測試 (可以使用 Copilot 的 /tests 功能協助產生)。

  • 執行程式碼並測試所有功能。
  • 檢查是否有錯誤訊息或非預期的行為。
  • 使用瀏覽器的開發人員工具或伺服器日誌來追蹤問題。

6. 優化與重構 (可選) 在基本功能完成後，可以要求 Copilot 優化程式碼的效能、可讀性或可維護性。
  • 要求 Copilot 找出效能瓶頸並提出改進建議。
  • 要求 Copilot 重構程式碼以提高可讀性。
  • 要求 Copilot 遵循特定的設計模式。
