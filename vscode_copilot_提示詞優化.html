VSCode Copilot 提示詞優化指南

本指南幫助您撰寫更有效的提示詞，讓 GitHub Copilot 在 VSCode 中提供更準確、更有用的幫助。

提示詞的基本原則

撰寫高效提示詞的關鍵原則：

  • 明確具體：清晰描述您想要完成的任務
  • 提供上下文：包含相關背景資訊
  • 指定格式：說明您希望收到的回應格式
  • 分步驟：對複雜任務，將其分解為步驟
  • 示例說明：提供具體示例說明您的需求

提示詞結構優化

1. 任務描述部分

基本結構

我想要 [具體動作]，功能需求是 [詳細描述功能]。

優化後

幫我 [具體動作]，具體要求如下：
1. [需求 1]
2. [需求 2]
3. [格式要求或輸出格式]
請使用 [程式語言/框架] 實現。

2. 環境與條件說明

基本結構

幫我寫個程式。

優化後

我正在使用 [框架/庫 版本] 開發一個 [項目類型] 應用。
目前環境：
- Node.js: v18.x
- 資料庫: MongoDB
- 前端框架: React 18

請幫我實現 [具體功能]，需要考慮 [特定環境限制/版本要求]。

3. 功能詳細說明

基本結構

做個用戶登錄。

優化後

請實現一個用戶登錄功能，包含以下要點：
1. 支持郵箱和密碼登錄
2. 實現記住我功能
3. 登錄失敗時顯示具體錯誤信息
4. 添加登錄嘗試次數限制（5次後鎖定10分鐘）
5. 防止SQL注入攻擊

技術要求：使用 Express.js 後端，React 前端，JWT 做身份驗證。

數據模型如下：
[描述數據模型或提供示例]

提示詞技巧進階篇

1. 具體情境設定

透過設定具體的專案背景和角色，讓 Copilot 能更精準地理解您的需求。 
角色：你是一位資深的前端架構師
項目：我們正在開發一個高流量的電子商務網站
任務：請為商品詳情頁面設計組件結構，需要考慮：
1. 組件重用性
2. 狀態管理方案
3. 性能優化策略
4. 適配移動端與桌面端
5. 支援國際化

請提出架構建議和關鍵代碼示例。

2. 迭代式提問

將複雜問題拆分為多個步驟，逐步引導 Copilot。

步驟 1：基本需求說明

我需要開發一個文件處理系統，能支援多種文件格式（PDF、Word、Excel）的上傳、預覽和轉換。請先幫我分析這個項目的技術選型。

步驟 2：深入特定問題

基於你推薦的技術棧，請幫我實現文件上傳功能，包含：
1. 拖拽上傳
2. 檔案大小和類型驗證
3. 上傳進度顯示
4. 斷點續傳功能

步驟 3：處理特殊情況

對於上傳的PDF檔案，我需要實現線上預覽和文本提取功能。請提供實現方案和關鍵代碼。

3. 要求解釋與教學

請求 Copilot 不只提供解決方案，還要解釋背後的原理，幫助您學習。 
請幫我實現一個 React 中的自訂 Hook 來處理無限滾動，並請：
1. 解釋這個 Hook 的設計思路
2. 標註關鍵部分的代碼並說明其作用
3. 討論潛在性能問題和優化方向
4. 提供一個使用示例

常見錯誤與改進

常見錯誤 改進建議
提示太籠統 添加具體需求、限制條件和預期結果
缺少關鍵信息 提供技術環境、版本信息、數據結構等
需求不明確 使用數字列表明確標記每個需求點
未指定輸出格式 明確說明需要的輸出形式（代碼、解釋、步驟等）
過於複雜的單次請求 將複雜問題拆分為多個連續的簡單問題

總結與最佳實踐

高效的 Copilot 提示詞應該：

  • 具體明確 - 描述清晰的目標和要求
  • 結構化 - 使用標題、編號列表組織內容
  • 提供上下文 - 包含必要的背景信息和技術環境
  • 指明限制 - 明確技術限制和特殊要求
  • 引導思考 - 要求解釋和教學，而不只是代碼輸出

重要提醒：

即使使用最優化的提示詞，請始終審查 GitHub Copilot 生成的代碼，確保其符合您的需求、安全標準和最佳實踐。Copilot 是工具，不能替代專業的開發判斷。
